# Grove v0.1 — Skeleton

## Goal

A runnable Rust binary that reads a config file, lists real repos and their worktrees in a sidebar, and lets you navigate between them. No live data, no terminals, no diff, no PR status. This phase exists to prove the layout, rendering, and app-data/config structure end-to-end so all later phases plug in cleanly.

## Acceptance criteria

- `cargo run` launches the TUI inside the user's terminal.
- First launch with no config: prints a friendly "no config found" hint showing the exact file path to create, then exits.
- With a config file present: renders the sidebar populated with repos from config and worktrees discovered via `git worktree list --porcelain`.
- Arrow/`j`/`k` moves the sidebar cursor; `h`/`l` collapses/expands repos; `Enter` "activates" a worktree (updates state, main pane shows which worktree is active).
- `?` shows a help overlay; `q` quits.
- App state (last-active worktree, expanded repos) is persisted to a state file and restored on next launch.
- Lightweight rendering test using `ratatui::TestBackend` snapshot.

## App data locations

Use the [`directories`](https://docs.rs/directories/latest/directories/struct.ProjectDirs.html) crate for cross-platform correctness.

```rust
let dirs = ProjectDirs::from("", "", "grove").expect("home dir");
```

| Purpose | Path (macOS) | Path (Linux) | Path (Windows) |
|---------|-------------|-------------|----------------|
| Config  | `~/Library/Application Support/grove/config.toml` | `~/.config/grove/config.toml` | `%APPDATA%\grove\config\config.toml` |
| State   | `~/Library/Application Support/grove/state.toml`  | `~/.local/share/grove/state.toml` | `%APPDATA%\grove\data\state.toml` |
| Logs    | `~/Library/Caches/grove/grove.log`             | `~/.cache/grove/grove.log`     | `%LOCALAPPDATA%\grove\cache\grove.log` |

**Why config vs state:**
- **config.toml** is user-authored. It's the source of truth for which repos exist. Never overwritten by the app.
- **state.toml** is app-authored. It stores ephemeral UI state (last active worktree, which repos are expanded, focus zone). Restored on startup so the app resumes where the user left off.

Mixing these would mean every rewrite of state.toml stomps on user-edited comments/formatting in config.toml. Keeping them separate lets the user edit config freely while the app owns state.

### Atomic writes for state.toml

```rust
// pseudocode
let tmp = state_path.with_extension("toml.tmp");
std::fs::write(&tmp, serialized)?;
std::fs::rename(tmp, &state_path)?;  // atomic on POSIX
```

Prevents a corrupted state file if grove is killed mid-write.

## File formats

### `config.toml`

```toml
# ~/Library/Application Support/grove/config.toml
# (or $XDG_CONFIG_HOME/grove/config.toml on Linux)

[general]
default_base_branch = "main"

[[repos]]
name = "grove"
path = "/Users/sebas/dev/grove"

[[repos]]
name = "dotfiles"
path = "/Users/sebas/dotfiles"
base_branch = "master"   # optional override
```

### `state.toml`

```toml
# Written and owned by grove. Do not edit.
schema_version = 1

[ui]
active_worktree = "grove:feat-sidebar"   # RepoName:BranchSlug
focus_zone = "sidebar"

[ui.expanded]
grove = true
dotfiles = false
```

The `schema_version` field lets future releases migrate or discard incompatible state files.

## Data model

```rust
// src/model.rs

pub struct Repo {
    pub name: String,
    pub root_path: PathBuf,
    pub base_branch: String,
    pub worktrees: Vec<Worktree>,
}

pub struct Worktree {
    pub branch: String,
    pub path: PathBuf,
    pub is_primary: bool,   // true for the main checkout
}

// src/app.rs

pub struct AppState {
    pub repos: Vec<Repo>,
    pub ui: UiState,
    pub config_path: PathBuf,
    pub state_path: PathBuf,
    pub should_quit: bool,
}

pub struct UiState {
    pub active_worktree: Option<WorktreeId>,   // (repo_idx, worktree_idx)
    pub sidebar_cursor: SidebarCursor,
    pub expanded: HashMap<String, bool>,       // repo name → expanded?
    pub focus_zone: FocusZone,
    pub modal: Option<Modal>,
}

pub enum Modal {
    Help,
}

pub enum FocusZone {
    Sidebar,
    MainPane,
}

pub enum SidebarCursor {
    Repo(usize),
    Worktree { repo: usize, worktree: usize },
}
```

`WorktreeId` is an opaque `(usize, usize)` for v0.1 — good enough since the repo list is fixed at load time. Upgrade to a stable ID in v0.2 once mutation becomes possible.

## Worktree discovery

`src/git.rs` shells out to:

```
git -C <repo_path> worktree list --porcelain
```

and parses the output. Porcelain format is stable:

```
worktree /Users/sebas/dev/grove
HEAD abc123...
branch refs/heads/main

worktree /Users/sebas/dev/grove/../grove-feat-sidebar
HEAD def456...
branch refs/heads/feat/sidebar
```

Parse into `Vec<Worktree>`. First entry is the primary checkout. Branches are `refs/heads/*`; strip the prefix.

If `git` exits non-zero (e.g. not a git repo), log a warning via `tracing` and skip that repo — don't crash.

## Event loop

Single-threaded in v0.1. Blocking on `crossterm::event::read()`.

```rust
loop {
    terminal.draw(|f| ui::render(f, &app))?;

    let event = event::read()?;                 // blocking
    let msg = input::handle(event, &app);       // Event → AppMessage
    app.update(msg);                            // pure state transition
    if app.should_quit { break; }
}
```

`AppMessage` enum:

```rust
enum AppMessage {
    MoveCursor(Direction),
    ToggleExpand,
    ActivateWorktree,
    ToggleHelp,
    CycleFocus,
    Quit,
    NoOp,
}
```

Pure state transitions keep v0.1 testable without spinning up a real terminal — `update` takes an `AppMessage` and mutates `AppState` deterministically.

On state change (after every `update`), save state.toml asynchronously via a debounced write (write at most once per 500 ms, and always on quit). For v0.1 synchronous is fine — save on Quit only.

## CLI

```
grove 0.1.0
A TUI for managing git repos, worktrees, and the work inside them.

USAGE:
    grove [OPTIONS]

OPTIONS:
    --config <PATH>     Override config file path
    --print-paths       Print config/state/log paths and exit
    --init              Create an empty config file at the default location
    -h, --help          Print help
    -V, --version       Print version
```

`--init` writes a commented example config to the default config path (idempotent — refuses to overwrite). First-run UX: the "no config found" message points the user at `grove --init`.

## Testing

Three lightweight tests. No full end-to-end terminal automation.

### 1. Config round-trip
```rust
#[test]
fn config_parses_with_minimal_fields()
```
Serialize a `Config` with two repos → parse back → assert equal.

### 2. Worktree parser
```rust
#[test]
fn parses_git_worktree_list_porcelain()
```
Static fixture string of porcelain output → assert returned `Vec<Worktree>` matches.

### 3. Sidebar snapshot
```rust
#[test]
fn sidebar_renders_expected_layout() {
    let backend = TestBackend::new(60, 20);
    let mut terminal = Terminal::new(backend)?;
    let app = AppState::with_fixture_repos();
    terminal.draw(|f| ui::render(f, &app))?;
    insta::assert_snapshot!(terminal.backend().to_string());
}
```
Uses `insta` snapshot. First run writes `.snap`; subsequent runs diff against it. Catches accidental rendering regressions.

Skip tests for: event loop integration, input handling details, TOML error messages. Those are easy enough to verify by hand in v0.1 and will accrete tests naturally as they become load-bearing.

## Implementation order (pull requests)

Breaking v0.1 into commit-sized chunks, each independently reviewable:

1. **Project scaffolding.** Cargo.toml, empty `main.rs` that renders an empty frame and quits on `q`. Paths module. Snapshot test harness.
2. **Config loading.** `Config` struct, TOML parsing, `--init`, `--print-paths`, first-run message.
3. **Worktree discovery.** `git worktree list --porcelain` parser with fixture-based test. `AppState::load()` wires config → model.
4. **Sidebar render.** Static rendering of repos + worktrees; first snapshot test.
5. **Navigation.** `j/k/h/l/Enter` update `UiState`; cursor highlight.
6. **Main pane placeholder.** "Select a worktree" / "Active worktree" states.
7. **Help overlay.** `?` toggles modal; `Esc` closes.
8. **State persistence.** Save/restore state.toml; atomic write.
