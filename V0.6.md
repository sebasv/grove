# Grove v0.6 — Local Diff Viewer

## Goal

Show "local changes" (staged + unstaged) for the active worktree, with keyboard-driven file navigation and stage/unstage actions. Replaces the terminal view when active.

## Acceptance criteria

- `Alt+d` toggles the main pane between Terminal and Diff for the active worktree.
- Diff pane layout: 30-col file list on the left, remainder shows the diff of the focused file.
- File list has two groups:
  - `UNSTAGED` — modified / deleted files in working tree vs index.
  - `STAGED` — files in index vs HEAD.
  - Each row: `{glyph} {path} {+adds -dels}` (glyph = ~ modified, + new, - deleted).
- `j/k` / `Up/Down` navigate the focused file; wraps across groups.
- `Tab` switches focus between file list and diff content; `J/K` scroll diff.
- `s` on an unstaged file stages it; `u` on a staged file unstages it. List refreshes afterwards.
- Diff content renders with colors: context lines plain, `+` additions green, `-` deletions red, hunk headers dim.
- `Esc` or `Alt+t` returns to the terminal view.
- Empty diff ("no local changes") renders a hint.
- Diff refreshes automatically on RepoDirty events (reuses the v0.3 watcher).

## Data model

```rust
pub enum MainView { Terminal, Diff }

pub struct DiffState {
    pub files: Vec<DiffFile>,   // in list order: unstaged first, then staged
    pub focus: DiffFocus,       // List or Content
    pub cursor: usize,           // index into files
    pub scroll: u16,             // diff content scroll offset
    pub loading: bool,
    pub error: Option<String>,
}

pub struct DiffFile {
    pub path: PathBuf,
    pub staged: bool,
    pub adds: u32,
    pub dels: u32,
    pub kind: DeltaKind,  // Added | Modified | Deleted | Renamed
    pub hunks: Vec<DiffHunk>,  // lines grouped per hunk
}

pub struct DiffHunk {
    pub header: String,
    pub lines: Vec<DiffLine>,
}

pub struct DiffLine {
    pub kind: LineKind,  // Context | Add | Del
    pub content: String,
}
```

`DiffState` lives per worktree alongside terminals (or inside `WorktreeState`). Stored on `AppState` as `HashMap<WorktreeId, DiffState>` so switching worktrees preserves scroll.

## git2 usage

```rust
let repo = Repository::open(worktree.path)?;
let unstaged = repo.diff_index_to_workdir(None, None)?;
let staged = repo.diff_tree_to_index(&head_tree, None, None)?;
```

Walk each via `foreach`/`patch` API, accumulating DiffFile/DiffHunk/DiffLine. For stage/unstage we run `git add <path>` / `git restore --staged <path>` (via Command) — git2's index-writing is fiddlier and not worth implementing directly for v0.6.

## Input routing (focus=Main, view=Diff)

| Key | Action |
|-----|--------|
| `j`/`Down` | Next file (list focus) or scroll down (content focus) |
| `k`/`Up` | Prev file (list focus) or scroll up (content focus) |
| `Tab` | Switch focus List ↔ Content |
| `s` | Stage file (list focus, unstaged) |
| `u` | Unstage file (list focus, staged) |
| `Alt+d` / `Esc` | Back to Terminal view |
| `Alt+t` | Terminal view (also closes diff) |
| `Ctrl+Space` | Cycle focus (grove-wide) |

## Critical review (self)

1. **Big diffs perform poorly.** We eagerly materialise every line into owned strings. For a 50k-line diff this is hundreds of MB. Mitigation: lazy rendering (hunk-by-hunk) is a v1.x optimization; for v0.6 we accept and document.

2. **Stage/unstage via `git` subprocess.** Fewer edge cases than poking git2's index directly (mode bits, renames, binary). One subprocess per operation is fine at human-interaction speeds.

3. **Auto-refresh after stage/unstage.** We explicitly re-run the diff load after each action instead of relying on the watcher, because the index change may not arrive in notify's events (some platforms skip inode-only updates).

4. **Concurrent diff load while UI is mutating.** Diff load runs on blocking pool; result is posted via Event. If the user navigated away by the time it arrives, we drop it. Mitigation: attach the WorktreeId to the event and drop if worktree doesn't match current state or if the diff was invalidated.

5. **Diff view + active terminal.** When user toggles to Diff, the terminal keeps running in the background. On toggle back, it's there. Same model as v0.5 terminal-tab switching.

6. **File paths in diff may be non-UTF-8.** git2 exposes them as `&Path` — we stringify via `display()` which lossy-converts. Acceptable.

## Tests

- Unit: diff parsing on a fixture repo (create staged + unstaged scenarios; assert DiffFile counts).
- Unit: DiffState cursor navigation across group boundaries.
- No snapshot — diff rendering is large and visually sensitive; testing via unit tests covering the core state machine is higher leverage.
