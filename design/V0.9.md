# Grove v0.9 — Worktree Create / Remove

## Goal

Create and remove git worktrees from inside grove. Press `w` with the sidebar focused on a repo (or one of its worktrees) to open a "new worktree" modal; `W` on a worktree row opens a confirm-remove modal.

## Acceptance criteria

- `w` (sidebar focus) opens a modal prompting for a branch name. `Enter` creates the worktree; `Esc` cancels.
- Worktree creation runs `git -C <repo_root> worktree add -b <branch> <path> <base>`:
  - `base` = repo's `base_branch` (from config).
  - `path` = `<repo_root parent>/<repo_name>-<sanitised branch>` (sanitise = replace `/` with `-`).
  - On failure, the modal stays open with an inline error.
- `W` on a worktree row opens a confirm-remove modal. `y`/`Enter` confirms.
- Confirm-remove runs `git -C <repo_root> worktree remove <path>` (without `--force`) then re-lists worktrees.
- UI state is cleaned up after remove (terminals, diffs, cursor, active-worktree references, expanded map untouched).
- `W` on the primary checkout is rejected (can't remove the main worktree) with an inline error.

## Data model

```rust
pub enum Modal {
    Help,
    AddRepo(AddRepoModal),
    ConfirmRemoveRepo { repo_idx: usize },
    NewWorktree(NewWorktreeModal),
    ConfirmRemoveWorktree { worktree_id: WorktreeId },
}

pub struct NewWorktreeModal {
    pub input: TextInput,
    pub error: Option<String>,
    pub repo_idx: usize,
}
```

## Critical review (self)

1. **Path derivation collision.** Two branches `feat/x` and `feat-x` both sanitise to `repo-feat-x`. Mitigation: if the derived path exists, append `-2`, `-3`, etc. Edge case worth handling.

2. **User's repo layout preferences.** Hardcoding "sibling directory" limits where worktrees live. Mitigation: future config `worktree_root` per repo; for v0.9 the sibling-dir convention is fine.

3. **Branch name validation.** git will refuse invalid names; we surface the stderr.

4. **Removal safety.** `git worktree remove` errors out if the worktree is dirty; we show the stderr as-is. Not `--force` by default.

5. **Terminals in the removed worktree.** Must drop any Terminal entries keyed by that worktree's index. Indices shift after remove — same pattern as `remove_repo`.

## Tests

- Unit: path derivation with collision.
- Integration: create a temp repo, run "new worktree" programmatically, assert it appears in `list_worktrees`.
- Integration: remove the created worktree; assert it disappears.
