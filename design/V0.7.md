# Grove v0.7 — Branch Diff Mode

## Goal

Add a second diff mode: "this branch vs the repo's base branch" (three-dot comparison using the merge base). Complements v0.6's local mode.

## Acceptance criteria

- `m` inside the Diff view toggles between `Local` and `Branch` modes.
- Branch mode computes `diff(tree(merge_base(HEAD, base)), tree(HEAD))` — only shows changes unique to this branch, not merges from base.
- Base branch is `Repo.base_branch` from config, defaulting to `general.default_base_branch` ("main").
- Mode indicator visible in the diff pane title or header row.
- `s`/`u` are no-ops in Branch mode (nothing to stage); no error surfaces, just ignored.
- Switching mode re-spawns the diff refresh.

## Critical review (self)

1. **Base branch not found.** Some repos don't have `main`; could be `master` or missing entirely. `Repository::find_branch(base, Local)` errors → return empty Vec and surface no results (user sees "No branch-diff content"). Acceptable.

2. **Merge base when branches have no common ancestor.** Shouldn't happen in a single-repo worktree setup, but if it does, `graph_descendant_of` / `merge_base` errors and we treat as empty diff.

3. **Detached HEAD.** We only handle branch HEAD. Detached → skip worktree in list_worktrees (existing behavior), so not reachable here.

4. **Mode persistence.** Per-worktree mode preserved in DiffState. Switching worktrees preserves each worktree's preferred mode.

## Tests

- Integration: create a repo, commit on `main`, branch `feat`, commit on `feat`, `compute_branch_diff` shows only the `feat` commit's delta.
