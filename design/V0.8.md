# Grove v0.8 — GitHub PR + CI Badges

## Goal

When a worktree's branch has an open GitHub pull request, show a badge on its sidebar row summarising PR state and CI.

## Acceptance criteria

- If `GITHUB_TOKEN` is set, grove discovers the GitHub owner/repo from each repo's `origin` remote, polls every 30s for each worktree with a branch, and caches a `PrStatus` on the worktree.
- If `GITHUB_TOKEN` is **not** set, polling is skipped silently; no badge.
- PR-state glyphs, appended to existing status badges:
  - `●` green — open
  - `◐` yellow — draft
  - `✓●` dim green — merged
  - `●` dim — closed without merge
- CI rollup glyphs:
  - `✓` — all checks passed (not rendered separately if PR is clean; the ● is enough)
  - `⟳` yellow — checks running
  - `✗` red — any check failed
- No PR → no PR-state glyph (existing status badges unchanged).
- Network errors (rate limit, offline) drop the badge to "unknown" silently. Retry on next tick.

## Data model

```rust
// model.rs
pub struct PrStatus {
    pub number: u32,
    pub state: PrState,
    pub checks: ChecksRollup,
}

pub enum PrState { Open, Draft, Merged, Closed }
pub enum ChecksRollup { None, Pending, Passing, Failing }
```

## octocrab usage

```rust
let octo = octocrab::OctocrabBuilder::new()
    .personal_token(token)
    .build()?;

let prs = octo.pulls(owner, repo)
    .list()
    .state(State::Open)
    .head(format!("{owner}:{branch}"))
    .per_page(1)
    .send()
    .await?;
```

Then for checks:

```rust
let runs = octo.checks(owner, repo)
    .list_check_runs_for_git_ref(Commit(head_sha))
    .send()
    .await?;
```

Aggregate: Failing > Pending > Passing > None.

## Critical review (self)

1. **Rate limits.** 30s × worktrees adds up fast (50 worktrees = 100+ calls/min). Mitigation: shared token = 5000/hour; with a warm-cache approach (only refetch if PR metadata changed) we're within budget for typical users (≤ 20 worktrees). Skip rate-limit handling beyond silent drop.

2. **Remote URL parsing.** `git@github.com:owner/repo.git` vs `https://github.com/owner/repo(.git)?` vs `ssh://git@github.com/owner/repo.git`. Need to handle all three. Miss any? Render no badge (safe fallback).

3. **Non-github remotes.** Parsing returns None; polling skipped.

4. **No PR → "searched and found nothing" vs "error".** Distinguish via Option<PrStatus> — None means either no PR or lookup failed. Can't tell apart; user sees no badge either way. Acceptable.

5. **Shutdown.** Poller is a `tokio::spawn` future. Dropping the channel senders causes it to error on next `send`, exiting cleanly.

## Testing strategy

- Unit: owner/repo parser for ssh / https / ssh://.
- No integration: real GitHub API would be flaky and require network. Skip.
