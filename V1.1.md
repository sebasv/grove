# Grove v1.1 — Mouse Support

## Goal

Clicks and scroll wheel events drive the UI. Keyboard is still primary; mouse is convenience.

## Acceptance criteria

- Left click on a sidebar worktree row sets cursor + activates the worktree.
- Left click on a sidebar repo row moves cursor to it.
- Left click on the main pane sets focus to Main.
- Left click on a terminal tab switches to that tab.
- Scroll wheel in the main pane (terminal) triggers scrollback scroll (up = back, down = forward).
- Mouse events are not forwarded to the PTY in v1.1 (simpler).

## Implementation

- Enable mouse: `execute!(stdout, EnableMouseCapture)` on start, `DisableMouseCapture` on restore.
- Crossterm Mouse events arrive on the same key event stream.
- Layout rects are computed at render time; to route clicks we cache the most recent sidebar/main/tab bar rects after each draw.

## Critical review (self)

1. **Rect tracking.** Mouse clicks need to know where each element is. Track rects on `AppState` (updated during draw) or recompute layout in the event handler. The former is simpler; tolerate the write.

2. **Tracking click coordinates vs scroll.** Scroll wheel is `MouseEventKind::ScrollUp/Down`. Clicks are `Down(Left)`. Only react on `Down`, not `Up` (avoid double-handling).

3. **Focus cycle semantics.** Clicking on the sidebar while focused on main should set focus back to Sidebar. Implicit in our click handler (we route based on which pane was hit).
