# Grove v1.2 — User-Configurable Sidebar Keys

## Goal

Let users remap the top-level sidebar bindings via `config.toml`. Other keys (PTY forwarding, diff view, modals) stay fixed for 1.x.

## Acceptance criteria

- `[keys]` section in `config.toml` accepts string mappings for each of these actions: `quit`, `help`, `refresh`, `add_repo`, `remove_repo`, `new_worktree`, `remove_worktree`.
- Key spec strings accept: plain chars (`q`, `?`, `a`), `Ctrl+<x>`, `Alt+<x>`, `Shift+<X>`, named keys (`Enter`, `Esc`, `Up`, `F1`). Case-insensitive modifier names.
- Unknown key strings fail config load with a clear message.
- Unset action → defaults (the v1.1 keymap).
- Remapping to an existing key silently replaces it: last declaration wins in the default_keys dispatch.

## Data model

```rust
pub struct Keymap {
    pub quit: KeyChord,
    pub help: KeyChord,
    pub refresh: KeyChord,
    pub add_repo: KeyChord,
    pub remove_repo: KeyChord,
    pub new_worktree: KeyChord,
    pub remove_worktree: KeyChord,
}

pub struct KeyChord {
    pub code: KeyCode,
    pub mods: KeyModifiers,
}
```

## Critical review (self)

1. **Overlapping bindings.** User rebinds `q` → `quit` but another action still uses `q`. We don't detect overlap at load time; the first-match wins based on order of checks in default_keys. Document.

2. **Unreachable bindings.** User maps `Ctrl+Space` → `quit`, but that's intercepted globally for focus cycle. Noop silently. Document.

3. **Localised to sidebar.** Scope is narrow; terminal-forwarding keys and modal keys stay fixed. That's a clear boundary.

4. **Serde path.** `KeyChord` has a custom `FromStr`/`Display` for serde-string serialization. Parse failures during TOML load give a clear message.
