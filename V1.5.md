# Grove v1.5 — Optional tmux Backing for Session Persistence

## Goal

When `[general] tmux_backing = true` in `config.toml`, grove spawns each embedded terminal inside a persistent tmux session named deterministically from the worktree path. Quitting grove leaves the session alive; restarting reattaches.

## Acceptance criteria

- `[general] tmux_backing = true` switches terminal spawning to `tmux new-session -A -s grove-<hash>`.
- Session name = `grove-<16-char hex of worktree path>`; identical across grove runs.
- `tmux_backing = true` but tmux not installed → startup warning, fall back to direct shell spawn.
- Behaviour unchanged when `tmux_backing = false` (default).
- Relaunching grove against the same worktree attaches to the tmux session; shell history / running processes survive.

## Data model

```toml
[general]
default_base_branch = "main"
tmux_backing = false  # default; set to true to enable
```

## Implementation notes

- `Terminal::spawn` takes an optional tmux-session name. When set, it executes:
  `tmux new-session -A -D -s grove-<hash>`
  `-A` = attach if session exists; `-D` = detach others so we own the session.
- Session name hashing uses `std::collections::hash_map::DefaultHasher` — deterministic across runs for the same path.
- Detection: run `which tmux`. Cache in AppState.

## Critical review (self)

1. **Cross-grove-instance coordination.** Two grove processes running against the same worktree would both try to attach. `-D` forces other clients to detach; acceptable, but the "last grove wins" behaviour should be documented.

2. **tmux status bar inside the PTY.** Visible by default — adds noise. Polish later with `tmux set-option status off` on first attach.

3. **Session leak.** Sessions persist until tmux dies (at OS level). Users may accumulate old sessions for deleted worktrees. Document `tmux kill-session -t grove-*` as cleanup.

4. **Hash collision.** DefaultHasher isn't stable across Rust versions. For v1.5 we accept that: if the Rust version changes, grove reattaches to a "new" session. Shells in the old session are orphaned but still accessible via `tmux ls`.
